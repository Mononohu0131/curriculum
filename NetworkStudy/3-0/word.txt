IP:ネットワーク層で使用されるプロトコル。現在の主流はIPv4。
IPヘッダ：IPパケットに含まれるもの。宛先住所であるIPアドレスが格納されている。
ICMP：Internet Control Message protocolの意。ネットワーク上での通信状況の確認や通信エラー時のメッセージの送信が主な用途。
ICMPメッセージ：pingコマンドにて、宛先の機器に送信するもの。
IPアドレス：データの宛先を識別するために必要となるアドレス。
オクテット：IPv4アドレスを８ビットごとに区切った時の、８ビットごとの単位。
ネットワーク部：IPアドレスの、ネットワークを識別する部分のこと。
ホスト部：IPアドレスの、ネットワーク上のホスト（PC、ルータ等のネットワーク機器）を識別する部分のこと
クラス：IPv4アドレスのネットワーク部とホスト部の境界を決める際の考え方。
プライベートIPアドレス：同一LAN内で重複しなければ自由に割当ができる、社内や家庭内で使用できるアドレス。
グローバルIPアドレス：インターネット上で利用される、インターネット上で重複が許されないアドレス。
ネットワークアドレス：ホスト部のビット数が全て０のアドレスで、ネットワーク自体を指す。そのネットワークの先頭のアドレス。
ブロードキャストアドレス：ホスト部のビット数が全て１のアドレスで、同一ネットワーク上に接続されている全てのホストにパケットを送信するためのアドレス。そのネットワークの一番後ろのアドレス。
ループバックアドレス：ネットワークアドレスとしても使用禁止とされているネットワークの「127.0.0.0 / 8」のネットワークで、
このアドレス。自分自身を示す仮想的なIPアドレスです。
サブネット：大きなネットワークの中にある小さなネットワークのこと。
サブネットマスク：IPアドレスの、ネットワーク部とホスト部の境界を示すために使われるアドレス。
プレフィクス表記：サブネットマスクを利用した時のネットワーク部の長さの表記
デフォルトゲートウェイ：異なるネットワークに通信する際の、ネットワークの出入り口。
IPv6：IPv4アドレスが枯渇してきていることを受け、新しく作成されたプロトコルがバージョン６のIPであるもの。
サブネットプレフィクス：IPv6アドレスの構造の中の、IPv4であればネットワーク部分となるもの。
インターフェイスID：IPv6アドレスの構造の中の、IPv4であればホスト部分となるもの。
ルータ：異なるネットワークに通信する際に、中継役となる機器。
ルーティングテーブル：ルータがIPパケットを受信した際、データを宛先へ転送するために最適なルートの情報が登録されているところ。
ネクストホップ：宛先に向かうために経由する隣接している次のルータのこと
ARP：Address Resolution Protocol（アドレス解決プロトコル）。IPアドレスからMacアドレスを調べるプロトコル
ARPテーブル：ARPのやりとりによって判明したIPアドレスとMACアドレスの関係情報が登録されているところ。
コアルータ：大型の、高速なパケット転送を実現するルータ。
センタールータ：企業で中心的に利用されるネットワークを接続するために利用されるルータ。
エッジルータ：企業の拠点間WANネットワークで支社や事業所等の拠点ネットワークを接続するために利用されるルータ。
ブロードバンドルータ：家庭環境やSOHO環境でインターネット回線に接続する目的で利用されるルータ。
ルーティング：ルータが送信されてきたパケットを宛先に送信する際、最適なルートを選んでパケットを転送すること
直接接続：コンピュータが、ネットワークではなくケーブルによって別のコンピュータに直接接続されている状態のこと。
スタティックルーティング：管理者がルーティングテーブルにルート情報を手動で登録する方法。
ダイナミックルーティング：ルータ同士がルーティングテーブルの作成に必要な情報を自動でやり取りすること。
ディスタンスベクタ型：互いのルーティングテーブルの情報を交換する方式。
リンクステート型：それぞれのルータが自身のインターフェイスの情報を交換する方式。ルーティングテーブル自体ではなく、テーブルを作成する材料を交換するイメージ。
ハイブリッド型：基本的な動作はディスタンスベクタ型だが、欠点を補うためにリンクステート型の要素も取り入れた型。

◯トランスポート層○
コネクション型通信：データの送信をする前に相手と通信できるか確認し、コネクション確立してからデータの送信を行うこと。
ACK：相手からの要求を承認して応答すること
コネクション型レス通信：経路確保などの事前やり取り無しに、データ送信を開始するパケット通信の方式。
TCP：コネクション型のプロトコル。接続の確立、送信データの順序の確保、フロー制御などがある。Transmission Control Protcol
スリーウェイハンドシェイク：TCPにおいて、通信を開始する前にデータの送信元と宛先のホスト間で接続を確立する方法。通信の開始を相手に確認→相手からの承諾→通信開始
Telnet：リモートでサーバなどにログインする際に使用されるプロトコル。
HTTP：インターネット上のWebサイトを閲覧する際にサーバとクライアント間の通信で利用されるプロトコル。Hypertext Transfer Protocol
FTP：ファイル転送に利用されるプロトコル。
SMTP：メールの送信の利用されるプロトコル。Simple Mail Transfer Protocol
（TelnetからSMTPはアプリケーション層で使用される）
TCPヘッダ：アプリケーション層から渡された分割されたデータに付加されるもの。信頼性を確保するために必要な情報などを含む。
UDP：通信速度を優先するプロトコル。データの信頼性より通信速度を重視するアプリケーションで利用される。User Datagram Protcol
RTP：音声や映像などをリアルタイムで配送するプロトコル。Realtime Transport Protocol
RTCP：RTPなどを用いてデータをリアルタイム伝送する際に、送受信制御を行うプロトコル。
DHCP：IPアドレスの自動割当を行うプロトコル。Dynamic Host Configuration Protocol
TFTP：FTPを簡易化したアプリケーション層のプロトコル。Trivial File Transfer Protocol
RIP：ディスタンスベクタ型に分類される、すべての情報を含むルーティングアップデートを定期的に送るプロトコル。Routing Information Protocol
UDPヘッダ：アプリケーション層から渡されたデータに付加するもの。TCPヘッダと異なり、信頼性を確保するための情報を付与する必要がないため、処理の負荷が軽くなります。
